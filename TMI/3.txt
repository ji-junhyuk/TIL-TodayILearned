<프로그램 독학할 때 하기쉬운 실수 5가지>
1. 언제나 공부만 하고 실제로 만들어보지 않는 것
-프로그래밍은 뭔가를 만들고, 완성하고, 프로젝트를 하는 것이다. (내 자신의 코드로)
-현실적인 목표로. 계산기 만들기 같은 엄청 쉬운 것 부터,,웹 스크래퍼,,작은 비디오겡미...
2. 언제나 준비가 되지 않았다고 생각하는 것
"난 아직 충분히 배우지 못했어"
3. 초반에 너무 열심히 해버리는 것
:매일매일 프로그래밍 하는거다. 습관을 만들어주는 것
핵심 : 프로그래밍이 재밌어야 함
4. 체크 리스트만 보면서 공부하는 것
프로그래밍은 공부해야 할 일의 목록을 지우면서 하는게 아니다.
내가 만들고 싶은 프로젝트의 리스트를 만들어봐.
5. 데드라인 없이 공부하는 것 
독학 = 게으름에 빠지기 쉽다.
나를 압박하는 무언가를 만들자.


1. 목표를 확실하게 설정한다.
 : 네가 하고 싶은, 원하는 것을 설정해라.

2. 자신과의 약속을 한다.
 : 끝까지 다 들어보고 판단해라.

3. 노트에 직접 작성하며 듣는다.
 : 디지털 노트를 사용해라. Hashing and Cashing. 자신이 작성하는 노트야말로 자신만의 알고리즘의 꽃이 아닐까...

4. 직접 정리하고 찾아보면서 배운다.
 : 스스로 이해하고 생각을 확장해라.

5. 퀴즈나 과제는 직접 해보고 결과를 확인한다.
 : 게으른 자는 실제 할 줄 아는 것이 없다. 좀 직접 쳐봐라...
